# Export Guide for GitHub & Cursor

This guide explains how to export your AurisVoice backend with a clean, minimal setup.

## Option 1: Quick Export (Recommended)

### For GitHub:

```bash
# 1. Create a fresh directory
mkdir aurisvoice-backend-clean
cd aurisvoice-backend-clean

# 2. Copy only the essential files
cp /path/to/replit/server.js .
cp /path/to/replit/.env.example .
cp /path/to/replit/.gitignore .
cp /path/to/replit/README.md .
cp /path/to/replit/package.minimal.json package.json

# 3. Install clean dependencies
npm install

# 4. Test it works
npm run dev

# 5. Initialize git and push
git init
git add .
git commit -m "Initial commit: AurisVoice minimal backend"
git remote add origin your-repo-url
git push -u origin main
```

### For Cursor:

1. Create new folder: `aurisvoice-backend`
2. Copy these files from Replit:
   - `server.js`
   - `.env.example`
   - `.gitignore`
   - `README.md`
   - `package.minimal.json` â†’ rename to `package.json`
3. Run `npm install` in Cursor
4. Start coding!

## Option 2: Export Current Replit Project

If you want to export the entire Replit project:

```bash
# 1. Clone or download the Replit project

# 2. Replace package.json with the minimal version
cp package.minimal.json package.json

# 3. Remove the old node_modules
rm -rf node_modules package-lock.json

# 4. Clean install
npm install

# 5. Test
npm run dev
```

## Why This Approach?

Replit's package.json cannot be directly edited to prevent environment corruption. By using `package.minimal.json` as a template, you get:

- âœ… Clean, minimal dependencies (only 4 packages)
- âœ… Production-ready configuration
- âœ… No React, TypeScript, or frontend bloat
- âœ… Fast installs and small footprint
- âœ… Perfect for GitHub and Cursor

## Support

If you encounter any issues after export:

1. Verify Node.js version: `node --version` (should be >= 18.0.0)
2. Clear npm cache: `npm cache clean --force`
3. Reinstall: `rm -rf node_modules package-lock.json && npm install`
4. Check .env configuration if API features aren't working
